결론부터 말하면, 히라가나/가타카나 다음 단계는 “문자 퀴즈”에서 바로 N5 문법으로 뛰는 것보다, 단어 + 발음 + 아주 짧은 문장 단계로 넘어가는 게 좋다.

아직 완전히 다 외우지 못했어도 괜찮다. 오히려 앱에서 다음 단계 학습을 섞어야 기억이 빨리 고정된다.

추천 순서
1단계: 히라가나/가타카나 복습을 “단어 단위”로 바꾸기

지금은 아마 이런 식일 거야.

あ → a
カ → ka

다음은 이렇게 바꾸면 좋다.

ねこ → neko → 고양이
みず → mizu → 물
テレビ → terebi → TV
コーヒー → koohii → 커피

즉, 문자 하나를 맞히는 게 아니라 실제 단어 안에서 읽게 만드는 것.

이 단계가 중요한 이유는, 실제 일본어를 읽을 때는 글자 하나씩 해독하는 게 아니라 단어 단위로 읽기 때문이다.

앱 기능으로는:

문제: ねこ
선택지:
1. inu
2. neko
3. mizu
4. yama

또는:

문제: “고양이”
정답 입력: ねこ

정도가 좋다.

2단계: N5 핵심 단어 300~500개

N1이 최종 목표라도 시작은 N5 단어부터 가야 한다. 여기서 중요한 건 “시험 급수”라기보다, 일본어 문장을 만들 최소 재료가 필요하다는 점이다.

우선순위는 이런 식이 좋다.

범위	예시
명사	人, 水, 本, 学校, 先生
동사	行く, 見る, 食べる, 飲む
형용사	大きい, 小さい, 高い, 安い
시간	今日, 明日, 昨日, 朝, 夜
위치	上, 下, 中, 前, 後
숫자	一, 二, 三, 十, 百

처음부터 한자를 강하게 넣을 필요는 없지만, 한자를 아예 빼는 것도 별로 좋지 않다.

예를 들면 이렇게 저장하는 게 좋다.

{
  "kana": "みず",
  "kanji": "水",
  "romaji": "mizu",
  "meaningKo": "물",
  "level": "N5",
  "partOfSpeech": "noun"
}

화면에는 처음에는 みず 중심으로 보여주고, 익숙해지면 水도 같이 노출하면 된다.

3단계: 조사 퀴즈

히라가나/가타카나 다음으로 제일 빨리 익혀야 하는 건 조사다.

특히 이 6개부터:

は, が, を, に, で, の

예를 들면:

私は 水___ 飲みます。
1. は
2. を
3. に
4. の

정답: を

이런 식으로 앱에 넣으면 문법 공부가 너무 무겁지 않으면서도 실제 문장 구조가 보인다.

초기 문법은 이 정도면 충분하다.

A は B です
A じゃありません
A を 食べます
A に 行きます
A で 食べます
A の B
4단계: 짧은 문장 읽기

단어만 계속 외우면 지루하고, 실제 독해로 이어지지 않는다. 그래서 아주 짧은 문장을 넣는 게 좋다.

私は 学生です。
물을 마십니다.
学校に行きます。
これは本です。

문제 유형은 이렇게 만들 수 있다.

私は 学生___。

2. を
3. に
4. の

나는 물을 마십니다.

[飲みます] [水を] [私は]

私は 水を 飲みます。

한자 같은건 어떻게 하지? 일어에 한자가 많잖아
2초 동안 생각함

결론: 한자는 “따로 한 글자씩 대량 암기”하지 말고, 일본어 단어 안에서 같이 익히는 방식이 맞다.
일본어 한자는 중국어처럼 한 글자 = 한 발음으로 단순하지 않고, 음독/훈독이 여러 개라서 단독 암기 효율이 낮다.

예를 들면:

水
みず       물        ← 훈독
すい       수        ← 음독, 水曜日(すいようび)

水 = sui 이렇게 외우면 실제로 水を飲む를 못 읽는다.
水 = みず / 水曜日 = すいようび처럼 단어 단위로 묶어서 외워야 한다.

앱에서는 이렇게 처리하는 게 좋다
1. 처음에는 kana 중심, kanji는 보조 노출

초기 단어 카드:

みず
水
물

처음부터 문제를 이렇게 내면 부담이 크다.

水 → ?

대신 처음에는 이렇게:

문제: みず
뜻: 물

아래에 작게 표시:
水

즉, 읽기는 kana로 먼저 안정화하고, 한자는 계속 눈에 노출시키는 방식.

2. 어느 정도 익숙해지면 “한자 인식 문제” 추가

다음 단계에서는 한자를 보고 뜻을 맞히게 한다.

問題: 水

1. 물
2. 불
3. 산
4. 사람

이건 읽기보다 쉽다.
초기 목표는 한자를 보고 의미를 알아보는 것이다.

순서는 보통 이게 좋다.

1. kana → 뜻
2. 뜻 → kana
3. kanji → 뜻
4. kanji → reading
5. 문장 안에서 kanji 읽기
3. 한자 단독 카드도 만들되, 메인은 단어 카드여야 함

한자 자체 정보는 필요하다. 다만 중심 데이터는 word가 되어야 한다.

예를 들어 水 한자 카드:

{
  "id": "kanji_water",
  "kanji": "水",
  "meaningKo": "물",
  "onyomi": ["スイ"],
  "kunyomi": ["みず"],
  "level": "N5"
}

하지만 실제 학습 단어는 따로 있어야 한다.

{
  "id": "word_mizu",
  "kana": "みず",
  "kanji": "水",
  "meaningKo": "물",
  "level": "N5"
}
{
  "id": "word_suiyoubi",
  "kana": "すいようび",
  "kanji": "水曜日",
  "meaningKo": "수요일",
  "level": "N5"
}

이렇게 해야 水의 여러 읽기를 자연스럽게 흡수할 수 있다.

핵심 구조

앱 데이터 모델은 대충 이렇게 가면 된다.

type Word struct {
	ID           string
	Kana         string
	Kanji        string
	MeaningKo    string
	Level        string
	PartOfSpeech string
	ExampleIDs   []string
}

type Kanji struct {
	ID        string
	Char      string
	MeaningKo string
	Onyomi    []string
	Kunyomi   []string
	Level     string
	WordIDs   []string
}

관계는 이렇게 보면 된다.

Kanji 1개 → 여러 Word에서 사용됨

水
 ├─ 水 / みず / 물
 ├─ 水曜日 / すいようび / 수요일
 └─ 水道 / すいどう / 수도

즉, Kanji는 사전/참조 데이터이고, 실제 학습 단위는 Word다.

앱 기능으로는 이런 단계가 좋다
v0.2: 단어 중심 한자 노출
문제: みず
정답: 물
보조 표시: 水

또는 정답 후에:

みず = 水 = 물

이 단계에서는 한자를 틀리게 만들지 말고, 그냥 자주 보게 만든다.

v0.3: 한자 의미 인식
問題: 山

1. 산
2. 강
3. 불
4. 나무

목표는 “읽기”보다 “뜻 인식”.

v0.4: 한자 읽기 문제
問題: 水

1. みず
2. ひと
3. やま
4. ほん

단, 여기서 조심해야 한다.
水는 문맥에 따라 みず일 수도 있고 すい일 수도 있다.

그래서 단일 한자보다 단어 단위 문제가 더 좋다.

問題: 水曜日

1. みずようび
2. すいようび
3. すいよび

한자 학습 순서

처음부터 상용한자 전체를 보려고 하면 망한다.
JLPT 기준으로 가는 게 낫다.

대략:

N5: 약 100자
N4: 약 300자 누적
N3: 약 650자 누적
N2: 약 1000자 누적
N1: 약 2000자 수준

정확한 숫자는 자료마다 조금 다르지만, 개발 로드맵 기준으로는 이 정도 감각이면 충분하다.
